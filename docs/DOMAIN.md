# Домен: Подготовка учебных материалов преподавателем

## Бизнес-контекст

Преподаватель готовится к занятию. Ключевая задача — создать презентацию, которая:
- Соответствует учебному плану
- Содержит все ключевые темы
- Связана с учебными материалами курса

Сейчас преподаватель тратит **2–4 часа** на создание презентации вручную. Это отнимает время от других задач: проверки домашних заданий, ответов на вопросы студентов, разработки новых материалов.

## Бизнес-задача

**Что нужно:** Ускорить подготовку презентаций без потери качества и согласованности с планом.

**Кто:** Преподаватель (или ассистент)

**Когда:** Перед каждым занятием

**Почему важно:**
- Экономия времени преподавателя
- Снижение ошибок
- Единый стиль презентаций в рамках курса

## Ключевые сущности

### План занятия (Lesson Plan)
- **Что это:** Документ, описывающий структуру занятия
- **Содержит:** Тему, цели, список разделов, ключевые материалы
- **Формат:** Текстовый документ (Markdown)
- **Владелец:** Преподаватель

### Презентация (Slide Deck)
- **Что это:** Набор слайдов для показа на занятии
- **Содержит:** Заголовки, тезисы, ссылки на материалы
- **Состояния:** Черновик -> Проверен -> Готов к показу
- **Создает:** Преподаватель (с помощью системы)

### Учебные материалы (Learning Materials)
- **Что это:** Ссылки на статьи, видео, книги
- **Связаны с:** Планом занятия и презентацией
- **Источник:** Библиотека курса, внешние ресурсы

## Бизнес-правила

1. **Презентация не может быть опубликована без проверки преподавателем**
   - Система создает только черновик
   - Финальное одобрение — за преподавателем

2. **Все ключевые темы из плана должны быть отражены в презентации**
   - Каждый раздел плана → отдельный слайд
   - Нельзя пропустить тему

3. **Ссылки на материалы должны быть актуальны**
   - Система копирует ссылки из плана
   - Преподаватель проверяет доступность

4. **Презентация должна быть в едином стиле курса**
   - Используются общие шаблоны оформления
   - Логотип, цветовая схема, шрифты

## Ограничения (бизнес-контекст)

1. **Модуль не заменяет преподавателя** — только помогает
2. **Качество контента зависит от качества плана** — чем детальнее план, тем лучше презентация
3. **Финальная презентация всегда проверяется человеком** — автоматизация не отменяет ответственность
4. **Модуль не создает новый контент** — только структурирует существующий
